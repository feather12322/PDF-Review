# 部署指南

## 一、本地开发环境部署

### 1. 后端部署

#### 步骤 1: 安装 Python 依赖

```bash
cd backend
python -m venv venv

# Windows
venv\Scripts\activate

# Linux/Mac
source venv/bin/activate

pip install -r requirements.txt
```

#### 步骤 2: 配置环境变量

```bash
cp .env.example .env
```

编辑 `.env` 文件，填入配置：

```env
# 使用 OpenAI
OPENAI_API_KEY=sk-your-api-key-here
OPENAI_BASE_URL=https://api.openai.com/v1

# 或使用通义千问（阿里云）
# DASHSCOPE_API_KEY=your-dashscope-key

# Redis（可选）
REDIS_HOST=localhost
REDIS_PORT=6379
```

#### 步骤 3: 启动后端服务

```bash
python main.py
```

服务将在 `http://localhost:5000` 启动

### 2. 前端部署

#### 步骤 1: 安装 Node.js 依赖

```bash
cd frontend
npm install
```

#### 步骤 2: 启动开发服务器

```bash
npm run serve
```

应用将在 `http://localhost:8080` 启动

## 二、生产环境部署

### 方案一：阿里云 Serverless + GitHub Pages

#### 后端部署到阿里云函数计算

1. **安装 Serverless Devs**

```bash
npm install -g @serverless-devs/s
```

2. **配置阿里云账号**

```bash
s config add
```

按提示输入 AccessKey ID 和 AccessKey Secret

3. **创建 s.yaml 配置文件**

在 `backend` 目录创建 `s.yaml`:

```yaml
edition: 1.0.0
name: resume-analyzer
access: default

services:
  resume-api:
    component: fc
    props:
      region: cn-hangzhou
      service:
        name: resume-service
        description: 智能简历分析服务
      function:
        name: resume-function
        description: 简历解析和匹配
        runtime: python3.9
        codeUri: ./
        handler: main.handler
        memorySize: 512
        timeout: 60
        environmentVariables:
          OPENAI_API_KEY: ${env(OPENAI_API_KEY)}
```

4. **部署**

```bash
cd backend
s deploy
```

部署成功后会返回函数的 HTTP 触发器地址

#### 前端部署到 GitHub Pages

1. **修改 API 地址**

编辑 `frontend/.env`:

```env
VUE_APP_API_URL=https://your-fc-domain.com/api
```

2. **构建生产版本**

```bash
cd frontend
npm run build
```

3. **部署到 GitHub Pages**

方法一：使用 gh-pages 工具

```bash
npm install -g gh-pages
gh-pages -d dist
```

方法二：手动推送

```bash
cd dist
git init
git add -A
git commit -m 'deploy'
git push -f git@github.com:yourusername/yourrepo.git master:gh-pages
```

4. **配置 GitHub Pages**

- 进入仓库 Settings > Pages
- Source 选择 `gh-pages` 分支
- 保存后访问 `https://yourusername.github.io/yourrepo`

### 方案二：Docker 容器化部署

#### 1. 创建 Dockerfile

**后端 Dockerfile** (`backend/Dockerfile`):

```dockerfile
FROM python:3.9-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 5000

CMD ["python", "main.py"]
```

**前端 Dockerfile** (`frontend/Dockerfile`):

```dockerfile
FROM node:16 as build

WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

#### 2. 创建 docker-compose.yml

```yaml
version: '3.8'

services:
  backend:
    build: ./backend
    ports:
      - "5000:5000"
    environment:
      - OPENAI_API_KEY=${OPENAI_API_KEY}
      - REDIS_HOST=redis
    depends_on:
      - redis
    volumes:
      - ./backend/uploads:/app/uploads

  frontend:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend

  redis:
    image: redis:alpine
    ports:
      - "6379:6379"
```

#### 3. 启动服务

```bash
docker-compose up -d
```

### 方案三：传统服务器部署

#### 后端部署（使用 Gunicorn）

1. **安装 Gunicorn**

```bash
pip install gunicorn
```

2. **启动服务**

```bash
gunicorn -w 4 -b 0.0.0.0:5000 main:app
```

3. **配置 Nginx 反向代理**

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location /api {
        proxy_pass http://localhost:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location / {
        root /var/www/frontend/dist;
        try_files $uri $uri/ /index.html;
    }
}
```

#### 前端部署

```bash
cd frontend
npm run build
sudo cp -r dist/* /var/www/frontend/dist/
```

## 四、性能优化建议

### 1. 启用 Redis 缓存

- 安装 Redis: `sudo apt install redis-server`
- 配置环境变量中的 Redis 连接信息
- 缓存可减少 80% 的重复计算

### 2. 使用 CDN

- 将前端静态资源部署到 CDN
- 加速全球访问速度

### 3. 配置 Nginx 缓存

```nginx
proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=api_cache:10m max_size=1g;

location /api {
    proxy_cache api_cache;
    proxy_cache_valid 200 1h;
    proxy_pass http://localhost:5000;
}
```

### 4. 数据库优化

- 使用 PostgreSQL 或 MySQL 替代内存存储
- 添加索引提升查询速度

## 六、常见问题

### Q1: CORS 跨域问题

确保后端已配置 CORS:

```python
from flask_cors import CORS
CORS(app)
```

### Q2: 文件上传失败

检查：
- 文件大小限制
- 上传目录权限
- Nginx 配置的 `client_max_body_size`

### Q3: AI API 调用超时

- 增加超时时间
- 使用异步调用
- 实现重试机制

## 七、安全建议

1. **API Key 保护**
   - 不要将 API Key 提交到代码仓库
   - 使用环境变量或密钥管理服务

2. **文件上传安全**
   - 限制文件类型和大小
   - 扫描恶意文件
   - 使用随机文件名

3. **HTTPS**
   - 生产环境必须使用 HTTPS
   - 配置 SSL 证书

4. **访问控制**
   - 添加用户认证
   - 实现 API 限流
   - 记录操作日志
